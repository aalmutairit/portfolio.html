SYS.MWAN / FEE-CALC-01
Licensing Fee Calculation Engine
Web App Arabic RTL Rules Engine
The Problem
At the National Center for Waste Management, licensing fees were calculated manually per application — roughly two minutes each, with errors affecting about 1 in 20 submissions. Each activity type carries its own regulatory fee structure, so the calculation meant cross-referencing fee tables by hand, every time.
What I Built
An Arabic-first web application that encodes the full regulatory fee schedule — every activity type with its own rate structure and duration-based rules — behind a deliberately simple two-field interface. The complexity lives in the rules engine, not on the user: pick the activity and the license duration, and the correct fee is computed instantly, the same way every time.
2min → <30s
Per-application processing time — with the prior 5% manual error rate eliminated, because every calculation follows the encoded fee schedule exactly.
A two-field front end concealing the full fee-schedule logic — Arabic-first, mobile-optimized.
